# QML Gap Analysis Pipeline

A local LLM-powered pipeline that compares automotive instrument cluster **specification documents** against **QML source code** to identify implementation gaps.

All processing runs on-premise using [Ollama](https://ollama.com/) — no data leaves your machine.

---

## Overview

Given a set of Japanese specification documents (converted to Markdown) and a QML source codebase, this pipeline:

1. Generates analysis prompts tailored to your spec format (via llama)
2. Statically analyzes the QML project structure
3. Indexes all documents into a local vector store (ChromaDB)
4. Runs section-by-section gap analysis for each spec
5. Outputs a report: **implemented / missing / extra** per spec document

## Gap Analysis — Quick Start

```bash
# Run the full pipeline with one command
python scripts/run_gap_analysis.py

# Override QML path without editing config
python scripts/run_gap_analysis.py --qml D:/path/to/qml

# Skip steps already completed
python scripts/run_gap_analysis.py --skip-analyze --skip-index

# Custom output directory
python scripts/run_gap_analysis.py -o reports/sprint42/
```

### Pipeline Steps

| Step | Script | Output |
|------|--------|--------|
| 1 | Ollama health check | — |
| 2 | `run_generate_prompts.py` | `prompts/compare.md`, `prompts/analyze.md` |
| 3 | `run_analyze.py` | `analysis/{timestamp}/` |
| 4 | `run_rag.py index` | `.rag_store/` (ChromaDB) |
| 5 | `run_rag.py compare` | `gap_report/{timestamp}/` |
| 6 | Summary print | console |

> Step 2 is skipped automatically if prompt files already exist.
> Delete `prompts/*.md` to force regeneration.

## Individual Scripts

### Convert spec documents to Markdown
```bash
python scripts/run_convert_to_md.py /path/to/docx/folder -o specs/
```

### Analyze QML project only
```bash
python scripts/run_analyze.py /path/to/qml/src
# Skip LLM summarization (faster, sufficient for gap analysis)
python scripts/run_analyze.py /path/to/qml/src --no-llm
```

### RAG index and query
```bash
# Index specs + analysis + QML source
python scripts/run_rag.py index specs/ analysis/{timestamp}/ --qml /path/to/qml/src

# Ask a question
python scripts/run_rag.py ask "What is the implementation status of feature X?"

# Run gap analysis only (after indexing)
python scripts/run_rag.py compare --specs specs/ -o gap_report/
```

### Regenerate prompts
```bash
# Let llama read your spec files and write optimized prompts
python scripts/run_generate_prompts.py --all --specs specs/

# Regenerate one prompt type
python scripts/run_generate_prompts.py --type compare --specs specs/
python scripts/run_generate_prompts.py --type analyze
```

### Auto-map specs to QML modules
```bash
python scripts/run_generate_map.py specs/ analysis/{timestamp}/ --llm
```

### Compare Excel files
```bash
python scripts/run_compare_xlsx.py data/old.xlsm data/new.xlsm \
  -s "Sheet1" --start 8 J K M --key J
```

### 3-Agent QML code generation (separate purpose)
```bash
python scripts/run_all.py
```

## How Prompts Work

`prompts/compare.md` and `prompts/analyze.md` are **generated by llama**, not written by hand.

The generator (`run_generate_prompts.py`) samples 3 random spec files, asks llama to analyze their format (CAN signal tables, state machine logic, EOL conditions), and writes a system prompt optimized for that content type.

This means the prompts adapt to your actual spec format automatically.

## Architecture — Gap Analysis Flow

```
specs/*.md (43 files)          QML source code
      │                              │
      ▼                              ▼
  Section chunking              Static parse
  (## 基本動作 sections)        (functions, signals,
      │                          properties, bindings)
      │                              │
      └──────────┬───────────────────┘
                 ▼
          ChromaDB (local)
          nomic-embed-text embeddings
                 │
                 ▼
         llama3.1:8b (local Ollama)
         per-section: "Is this requirement implemented?"
                 │
                 ▼
         gap_report/
         implemented / missing / extra
```
